## 相談事項
上記2にも記したが、GitHubの戦略を決めたいです。私は、Git Flowからrelease branchesを抜いたもので良いかなと考えています。理由としては、主に次の2点が上げられます。

1. 今回の開発の流れが前半で必須機能を実装しきって、後半で追加機能を実装する、という流れ上、main branchにはver1.必須機能がバグなしで動くversion、ver2.追加機能を加えた状態でバグなしで動くversionという形で保持しておきたいと考えたため。
2. release branchesに関しては、これを付けるとbranchの数が増えるため管理が大変だと考えたため。また、GitHub Flowを見てみると、mainにすぐ統合、という形をとっており、develop branchは残した方が良いと思った一方、release branchesまで残すのは神経質すぎると考えたため。
