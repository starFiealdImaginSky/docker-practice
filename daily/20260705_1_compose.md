## Docker Composeの書き始め

Docker Composeとは、複数のDockerコマンドを1つの設定ファイルにまとめて実行できる仕組みである。簡単な例題として、FastAPIだけをComposeで起動してみる。

```yaml
services:
  web:
    build: .
    ports:
      - "8000:8000"
```
* 1行目：このプロジェクトで起動するコンテナ(サービス)の一覧を書く、という宣言。
* 2行目：webというサービス名でコンテナを列挙する。実は任意である。
* 3行目：docker build . を実行する。即ち、カレントディレクトリにあるDockerfileをビルドしている。
* 4行目：ホストとDockerコンテナ間でのネットワークのマッピングを指定する。

Dockerfileが不要な例を次に示す。
```yaml
services:
  python:
    image: python:3.13
    command: python --version
```
上の例は、イメージとして、公式が提供しているものを使っている。その際はそもそもビルドが必要ない。

Composeによる起動方法は
```bash
docker compose up --build
```
バックグラウンドで起動する場合には、上部コマンドの後ろに"-d"を追加すればよい。なお、--buildは、イメージのビルドから行うようにするコマンドであり、これがないとファイルの変更が更新されない。

また、docker composeについて起動したコンテナは人間にわかりやすい名前が自動で付く。
```bash
docker ps
```
で確認すると良い。

また、composeの停止は
```bash
docker compose down
```
で出来る。
## volumesについて
現在、main.pyを編集したらビルドをし直さなければならない。それは、Dockerfileの
```Dockerfile
Copy . . 
```
がビルド時点のファイルをイメージにコピーするという命令だからである。

ここで、volumesを用いることで、**ホストのフォルダとコンテナのフォルダを直接つなげるようにする。**

```yaml
services:
  web:
    build: .
    ports:
      - "8000:8000"

    volumes:
      - .:/app
```
volumes:以降は次のルールに則って書く。
```yaml
- <ホスト側のディレクトリ>:<コンテナ側のディレクトリ>
```
今回の場合は、ホスト側のディレクトリ：カレントディレクトリ、コンテナ側のディレクトリ：/appであるが、これを直接つなげるということであり、ホスト側：docker-practice/とコンテナ側：/appが同じものになる。

実際、main.pyを書き換えた後、docker compose down、docker compose upを叩けば、ビルドなしでコンテナ側に反映されている。

注意したいのは、この書き方は開発環境なら良いが本番環境ではしない方がよいということだ。なぜなら、Dockerの理念は「イメージだけで同じ環境を再現できること」である。"-.:/app"はホストのファイルに依存しているため、理念から外れている。そのため、開発中は効率を優先してvolumesを使うが、本番では再現性を優先してvolumesを外すべきである。

## command
commandはDockerfileのCMDをCompose側から上書きする機能である。
Dockerfileで
```Dockerfile
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```
に対応するcompose.yamlのcommandは
```yaml
services:
  web:
    build: .
    command: uvicorn app.main:app --host 0.0.0.0 --port 8000
```
もしくは、複数行で
```yaml
services:
  web:
    build: .
    command: > 
    uvicorn app.main:app
    --host 0.0.0.0
    --port 8000
```
と書ける。本番環境の設定をDockerfileのCMDに、開発環境の設定をComposeのcommandに書くという使い分けが可能であり、例えば、
```yaml
services:
  web:
    build: .
    command: >
      uvicorn app.main:app
      --host 0.0.0.0
      --port 8000
      --reload
```
とreloadを追加すれば、main.pyの変更を保存後、ブラウザをリロードするだけで変更が更新される。
