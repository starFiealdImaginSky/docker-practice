# 2026年07月04日　まとめ2.gitとGitHub
## 1. プロジェクトを作る

1. 20260704_1_環境構築.mdで整えたフォルダ構造を使う。
    ```bash
    cd ~/projects
    mkdir docker-practice
    cd docker-practice
    ```
    とコマンドを叩いて、docker-practiceフォルダに入る。
2. このフォルダを１個のプロジェクトとみなし、Gitでの管理を行う。そのためのコマンドを下に記す。
    ```bash
    git init
    ```
3. READMEを作る。折角なので、
    ```bash
    code .
    ```
    と叩いて、VScode上で行うことにする(GUI操作ができて楽)。
    docker-practiceフォルダ直下にREADME.mdファイルを作成、適当な文を入力すれば良い。(例：# Docker Practice)
4. Gitに登録する。
    ```bash
    git add README.md
    ```
    でステージングして(コミットの対象にするファイルを選ぶ)、
    ```bash
    git commit -m "コメント"
    ```
    でコミットできる。

5. ブラウザでGitHubを開き、New repositoryを押す。名前を付けれる場所は任意であるが、docker-practiceとかしておく。色々設定できるが、ローカル側でREADME.mdを作成しているので、Add READMEにチェックを入れないようにする。<br>
リポジトリが作れたら、
    ```
    git@github.com:ユーザー名/docker-practice.git 
    ```
    という一文がコピーできるボタンが現れるのでコピーをする。
6. Ubuntu(VScodeのターミナル)に戻り、次のコマンドを打つ。git@以降は5.でコピーしたものを用いると良い。
    ```bash
    git remote add origin git@github.com:ユーザー名/docker-practice.git
    ```
7. 最後に、
    ```bash
    git push origin main
    ```
    でを行うことで、ローカル側でコミットしたREADME.mdファイルが、GitHubにアップロードされる。

## 2.チーム開発におけるgit手順
1. まずは、リモート環境のプロジェクトをローカルに落とさなければならない。次のコマンドを叩く
    ```bash
    git pull origin main
    ```
2. ローカル側のmainを壊すのは良くないので、ブランチを作成し、そちらに移動してファイルをいじる。新しいブランチをnewBranchを名付ける。
    ```bash
    git branch newBranch
    git checkout newBranch
    ```
    なお、今あるブランチ一覧を見たい場合は、
    ```bash
    git branch
    ```
    で確認できる。
3. ファイルを書き換えたら、git add、git commitを行う。プロジェクト作成と同じ流れなので省略するが、タイミングとしては、1つの意味のある変更ごとがオススメらしい。
4. キリがよくなったらpushする。pushはコミットほど細かくなくてよい。チーム開発であることを意識して、次のコマンドを打つ。
    ```bash
    git push origin newBranch
    ```
    注意したいのが、mainブランチにpushをするのではなく、GitHub側に、ローカルで作ったブランチと同じブランチ名を作って、そこにpushする必要がある(コンフリクトを防ぐため)。
5. ブラウザでGitHubのリポジトリを開き、"Compare & pull request"をクリック。その後、タイトルや説明を書いて、Create Pull Requestボタンを押す。Pull Request画面でMerge pull request、Confirm mergeを押すことで無事GitHub側のmainブランチにアップロードされる。
6. ローカルでmainブランチに戻り、不要になったブランチを消しておく。また、ローカルも更新しておく。
    ```bash
    git checkout main
    git branch -d newBranch
    git pull origin main
    ```
